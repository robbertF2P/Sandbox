Platform 2.0 — Authentication & tenant user lifecycle

Design document (not implemented) · SandBox repo · Generated from platform-authentication-standard.md

1. Summary

  • One product binary for cloud SaaS and on-prem; behaviour driven by deployment profile and install config.
  • OIDC + PKCE for browser login; platform issues its own JWT with tid, sub, and roles.
  • One database per tenant — optional shared SQL Server instance, never a shared multi-tenant database.
  • Identity module (F2P.Identity / @f2p/identity) owns login accounts, roles, and tenant-admin user management.
  • Admin backoffice (separate) provisions tenants; tenant admins manage operational users who book hours and edit planning data.

2. Cloud authentication flow

StepActorAction
1BrowserUser opens https://{slug}.app.example
2EdgeResolve slug via control plane → tenantId + OIDC settings
3ShellRedirect to customer IdP with PKCE code_challenge
4IdPUser authenticates (MFA enforced by customer IdP)
5Identity APIExchange auth code; map IdP subject → platform user
6Identity APIIssue platform JWT (tid, sub, roles)
7Domain APIsValidate JWT; open connection to tenant's dedicated database

On-prem variant

Fixed host and tenantId from install config replace slug lookup. Same OIDC sequence; signing keys and user store may be customer-managed.

3. Tenant user lifecycle

User states

InvitedPre-provisioned by tenant admin; not yet signed inAdmin creates user with email + roles
PendingAwaiting first successful loginInvite sent or JIT record created
ActiveCan sign in and use product per rolesFirst OIDC / local login succeeds
DisabledLogin blocked; data retainedAdmin disable or seat reclaim
RetiredOffboarded; audit trail keptAdmin retires after project end

Login account vs Person

Legacy F2P separates User (authentication) from Person (workforce entity for hours and assignments). V2 preserves this:

  • Identity.User — login principal, roles, enabled/disabled
  • Resources.Person — who appears on timesheets and WBS assignments
  • Tenant admin flow should support create user + link Person in one wizard where hour booking is required
