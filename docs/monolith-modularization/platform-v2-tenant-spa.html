Platform 2.0 — Tenant SPA (single application)

Architecture supplement · SandBox F2pPlatform/web/ · Slides 9–12 in platform-architecture-overview.md

1. Summary

  • One tenant product SPA — f2p-shell is the only Angular app end users run for planning, hours, hour approvals, etc.
  • Bounded contexts are libs — lazy-loaded via loadChildren / loadComponent; not separate deployed apps.
  • One auth session — IdentityAuthService (providedIn: 'root') + shell authGuard + bearer interceptor on all /api/ calls.
  • One visual product — global @import '@floorganise/css'; shared navbar and tiles from @floorganise/ui.
  • Same origin — empty API base URL; dev proxy and production nginx forward /api and /hubs to the composed host.
  • Separate operator app — f2p-admin-shell is the platform console (tenant provisioning), not the tenant product.

4. Shell composition root

The shell is thin: bootstrap, router, global styles, and cross-cutting providers only.

// apps/f2p-shell/src/app/app.component.ts
@Component({
  selector: 'f2p-root',
  imports: [RouterOutlet],
  template: '<router-outlet />',
})
export class AppComponent {}
// apps/f2p-shell/src/app/app.config.ts
export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(routes),
    provideHttpClient(withInterceptors([correlationInterceptor, identityAuthInterceptor])),
    { provide: F2P_API_BASE_URL, useValue: '' },
  ],
};

5. Lazy routes — modules plug into one router

// apps/f2p-shell/src/app/app.routes.ts (excerpt)
{
  path: 'hour-approvals',
  canActivate: [authGuard],
  loadChildren: () =>
    import('@f2p/hour-approvals/feature-tasks').then(m => m.hourApprovalsRoutes),
},

Each context feature-* lib exports child routes and optional capability guards. Adding a context:

  1. ./scripts/scaffold-frontend-module.sh <Context>
  2. Register lazy route in app.routes.ts
  3. Add home tile in home-page.component.ts
  4. Pair with backend ./scripts/scaffold-module.sh <Context> + Program.cs

7. Monorepo layout

web/
├── apps/f2p-shell/              tenant product SPA
├── apps/f2p-admin-shell/        platform operator (separate)
└── libs/
    ├── shared/api-core/         correlation interceptor
    ├── shared/platform-events/  root SignalR client
    ├── identity/data-access/    auth service, guards
    └── <context>/
        ├── data-access/         API clients, feature guards
        └── feature-*/           routes, smart pages

SandBox uses TypeScript path aliases; monolith floor2plan-web uses Nx with identical layering. Nx boundary tags prevent cross-context imports while keeping one deployable SPA.

8. Same-origin deployment

Development: Angular dev server proxies /api and /hubs to localhost:5080.

Production: nginx serves the SPA bundle and proxies API/SignalR; try_files $uri $uri/ /index.html enables deep links like /hour-approvals.

9. Backend mirror

The frontend single-origin model matches one composed API host:

builder.Services.AddIdentityModule(builder.Configuration);
builder.Services.AddHourApprovalsModule(builder.Configuration);
app.MapIdentityModule();
app.MapHourApprovalsModule();
app.MapHub<PlatformEventsHub>("/hubs/platform-events");
